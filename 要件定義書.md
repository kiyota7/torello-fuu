## 1. はじめに

### 1.1 背景

チームや個人でタスクを管理する際、付箋やメモだけでは進捗状況の把握が難しく、
「未着手」「進行中」「完了」といった状態ごとの整理ができないという課題がある。
そこで、Trelloに代表されるカンバン方式(リストとカードによる視覚的なタスク管理)を
参考にした、シンプルなタスク管理Webアプリケーションを開発する。

### 1.2 目的

本ドキュメントは、開発するタスク管理アプリ「タスクボード」(以下、本システム)の
機能・仕様を明確にし、開発内容の認識をそろえることを目的とする。

### 1.3 対象読者

本システムの発注者(依頼者)、開発者、および課題の評価者。

### 1.4 想定発注シーン(ヒアリング内容)

本課題では、以下のような依頼を受けた想定で要件を整理する。

> 個人でカフェ開業に向けた準備を進めている依頼者から、「やることが多く、
> 何が終わっていて何が残っているのか一目で分からず困っている。付箋やメモ帳では
> 管理しきれないので、Trelloのような画面でタスクを整理したい。ただし専門知識がなくても
> 迷わず使えるシンプルなものがよい」という相談を受けた。

この相談内容から、以下の3点を本システムが満たすべき条件として設定する。

- 特別な操作説明を読まなくても直感的に使えること
- 「未着手・進行中・完了」のような状態でタスクを視覚的に整理できること
- 個人利用が主目的のため、アカウント登録等の複雑な仕組みを持たないこと

## 2. システム概要

### 2.1 システム名称

タスクボード(Trello風タスク管理アプリ)

### 2.2 システムの位置づけ

Webブラウザ上で動作する単一ページのタスク管理アプリケーション。
サーバーやデータベースを持たず、ブラウザ内にデータを保存する
スタンドアロン型のツールとする。

### 2.3 想定利用者

- 個人でタスクを整理したい学生・社会人
- 小規模なチームで簡易的にタスク状況を共有したい利用者

### 2.4 利用シーン

- 学校の課題や日々のToDoを「To Do」「進行中」「完了」のリストに分けて管理する
- リストを自由に追加し、プロジェクトごとの進捗管理に応用する

## 3. 機能要件

本システムが備える機能は以下の通りとする。

| 機能ID | 機能名 | 概要 |
|---|---|---|
| F-01 | リストの追加 | 「+ リストを追加」からリスト名を入力し、新しいリスト(列)を作成できる |
| F-02 | リスト名の編集 | リストのタイトル部分をクリックすると直接編集でき、変更内容が保存される |
| F-03 | リストの削除 | リスト右上の「✕」から、リストと配下の全カードを削除できる(確認ダイアログあり) |
| F-04 | カードの追加 | 各リスト下部の「+ カードを追加」からカード(タスク)を作成できる |
| F-05 | カード内容の編集 | カードのテキスト部分をクリックすると直接編集でき、変更内容が保存される |
| F-06 | カードの削除 | カードにカーソルを合わせると表示される「✕」から、確認なしで即時削除できる |
| F-07 | カードの移動(ドラッグ&ドロップ) | カードをドラッグし、任意のリストにドロップすることでカードの所属リストを変更できる |
| F-08 | データの自動保存 | リスト・カードの作成/編集/削除/移動内容は、ブラウザのlocalStorageに自動保存され、再訪問時も保持される |
| F-09 | 初期データへのリセット | 画面右上の「サンプルに戻す」から、保存データを削除しサンプルの初期状態に戻せる(確認ダイアログあり) |

### 3.1 機能対象外(スコープ外)とする項目

学校課題として簡易実装とするため、以下の機能は本システムの対象外とする。

- ユーザー登録・ログイン機能、複数ユーザーでの共同編集
- カードの期限日・担当者・ラベル・添付ファイルなどの詳細情報
- リストやカードの並び替え(同一リスト内での順序変更)
- サーバーサイドでのデータ保存、複数端末間でのデータ同期
- 検索・フィルタ・通知機能

## 4. 非機能要件

| 項目 | 内容 |
|---|---|
| 動作環境 | 最新版のGoogle Chrome、Microsoft Edge等のモダンブラウザ |
| 実行方法 | インストール不要。HTMLファイルをブラウザで開くだけで起動する |
| レスポンス性能 | 操作(追加・編集・削除・移動)に対して体感的な遅延なく画面へ反映されること |
| データ保存範囲 | データは利用端末のブラウザ内(localStorage)にのみ保存され、他端末とは共有されない |
| ユーザビリティ | 直感的なクリック操作・ドラッグ操作のみで、マニュアルなしに利用できること |
| セキュリティ | 個人情報や機密情報を扱わない前提とし、認証・暗号化等は実装しない |

## 5. 画面設計

### 5.1 画面一覧

本システムは以下の1画面のみで構成される。

| 画面ID | 画面名 | 概要 |
|---|---|---|
| S-01 | ボード画面 | リストとカードを表示・操作する唯一のメイン画面 |

### 5.2 画面構成要素

ボード画面は以下の要素で構成する。

- ヘッダー: アプリタイトル、「サンプルに戻す」ボタン
- リスト(複数): リスト名、カード一覧、「+ カードを追加」ボタン、リスト削除ボタン
- カード(各リストに複数): カード本文、カード削除ボタン
- 「+ リストを追加」: 画面右端に常に表示される、新規リスト作成用の入力欄

### 5.3 画面遷移

画面遷移は発生しない(単一画面内でのDOM操作により全ての機能を提供する)。

### 5.4 画面イメージ(ワイヤーフレーム)

簡易版のワイヤーフレームを以下に示す。実装時はこのレイアウトを基準とし、
配色・フォント等の詳細デザインは別途モックアップで確認する。

```
+----------------------------------------------------------------------+
|  タスクボード                                    [ サンプルに戻す ]  |
+----------------------------------------------------------------------+
|  To Do            |  進行中           |  完了            |           |
|  ------------      |  ------------     |  ------------     |  + リスト |
|  [カード1     ✕]  |  [カード3     ✕] |  [カード5     ✕] |    を     |
|  [カード2     ✕]  |  [カード4     ✕] |                  |    追加   |
|  + カードを追加    |  + カードを追加   |  + カードを追加   |           |
|  ------------      |  ------------     |  ------------     |           |
|  [ リスト削除 ✕ ] |  [ リスト削除 ✕ ]|  [ リスト削除 ✕ ]|           |
+----------------------------------------------------------------------+
```

- カードはマウスドラッグでリスト間を移動できる(F-07)
- リスト名・カード本文はクリックでその場編集ができる(F-02, F-05)
- 一番右の「+ リストを追加」列は常に画面右端に固定表示する

## 6. データ設計

サーバー・データベースを持たないため、ブラウザのlocalStorage上に
以下の構造のJSONデータとして保持する。

### 6.1 エンティティ関連

```
Board (1) ── (多) List (1) ── (多) Card
```

### 6.2 データ項目

**List(リスト)**

| 項目名 | 型 | 説明 |
|---|---|---|
| id | 文字列 | リストを一意に識別するID |
| title | 文字列 | リスト名(例: To Do, 進行中, 完了) |
| cards | 配列(Card) | このリストに属するカードの一覧 |

**Card(カード)**

| 項目名 | 型 | 説明 |
|---|---|---|
| id | 文字列 | カードを一意に識別するID |
| text | 文字列 | カードの内容(タスク名) |

### 6.3 保存形式

- 保存先: ブラウザのlocalStorage(キー名: `trello-lite-board`)
- 保存タイミング: リスト・カードに対する追加/編集/削除/移動が発生する都度、即時保存する

## 10. 制約事項

- 本システムはローカルブラウザにのみデータを保存するため、ブラウザのキャッシュ削除や
  端末変更によりデータが失われる可能性がある。
- 複数人での同時編集には対応していない。
- 対応ブラウザ以外(古いブラウザ等)での動作は保証しない。

## 11. 今後の拡張案(参考)

学校課題としての現バージョンでは対象外とするが、発展課題として以下が考えられる。

- サーバーサイドと連携したデータの永続化・複数端末同期
- 複数ユーザーによる共同編集機能
