# ローカル開発環境セットアップ(バックエンド)

このマシンではHomebrewを使わず、sudo不要な方法でJava/Maven/Dockerを用意している。
別のマシンでセットアップする場合は、Homebrew等を使って同等のバージョンを用意すればよい。

## 導入済みのバージョン

| ツール | バージョン | 導入方法 |
|---|---|---|
| Java | Eclipse Temurin JDK 21 | tar.gz を `~/dev-tools/` に展開(インストーラ不要) |
| Maven | 3.9.9 | tar.gz を `~/dev-tools/` に展開(インストーラ不要) |
| Docker Desktop | 4.81.0 | 公式dmgからインストール |
| PostgreSQL | 16(Dockerコンテナ) | `backend/docker-compose.yml` |

`~/.zshrc` にJAVA_HOME・PATH(Java/Maven/Docker CLI)を追記済み。新しいターミナルであれば
`java` `mvn` `docker` コマンドがそのまま使える。

補足: `/usr/local/bin/docker` はインストーラーの一時マウントボリュームを指した壊れたシンボリック
リンクになっており、sudo権限がないため修復できない。そのため `~/.zshrc` で
`/Applications/Docker.app/Contents/Resources/bin` を直接PATHに追加する形で回避している。

なお、当初はPostgres.app(管理者権限不要でインストールできるアプリ)でPostgreSQLを動かしていたが、
Dockerが使えるようになったため、Docker Compose管理のPostgreSQLコンテナに置き換えた
(Postgres.appは停止済みで未使用)。

## PostgreSQLの起動・停止(Docker Compose)

```
cd backend

# 起動
docker compose up -d

# 状態確認
docker compose ps

# 停止
docker compose down

# 停止 + データも削除する場合
docker compose down -v
```

- 接続先: `localhost:5432`、DB名 `taskboard`
- 接続ユーザー: `taskboard` / パスワード: `taskboard`(開発環境専用の簡易パスワード)
- データはDocker named volume(`taskboard-db-data`)に永続化される

## バックエンドアプリの起動

```
cd backend
mvn spring-boot:run
```

起動後、`http://localhost:8080/api/health` にアクセスすると `{"status":"ok"}` が返る。

DB接続情報は `src/main/resources/application.properties` の
`DB_USERNAME` / `DB_PASSWORD` 環境変数(デフォルト値は上記のDocker Compose設定と一致)で
上書きできる。

## 現状のひな形に含まれるもの

- `AppUser` / `TaskList` / `Card` エンティティ([データベース設計書](データベース設計書.md)のテーブル定義と対応)
- 各エンティティの `JpaRepository`
- `SecurityConfig`: 暫定的に全リクエストを許可する設定(ログイン機能実装時に認証必須へ変更する)
- `/api/health`: 動作確認用のヘルスチェックエンドポイント
