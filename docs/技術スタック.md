# 技術スタック

タスク管理アプリ「タスクボード」の開発環境・使用技術。

## 全体構成

フロントエンド(React)とバックエンド(Spring Boot)を分離した構成とし、
データはPostgreSQLに永続化する。フロントエンドはバックエンドが提供するREST APIを介して
データの取得・更新を行う。

```
[ブラウザ] ⇄ React(フロントエンド) ⇄ REST API(JSON) ⇄ Spring Boot(バックエンド) ⇄ PostgreSQL
```

## フロントエンド

| 分類 | 内容 |
|---|---|
| 言語 | JavaScript / TypeScript |
| フレームワーク | React |
| ルーティング | React Router(ログイン・新規登録・ボード画面間の遷移用) |
| ビルドツール | Vite |
| API通信 | fetch APIによるバックエンドREST APIの呼び出し |

## バックエンド

| 分類 | 内容 |
|---|---|
| 言語 | Java |
| フレームワーク | Spring Boot |
| 認証 | Spring Security(セッション/Cookieベース。パスワードはbcryptでハッシュ化) |
| ビルドツール | Maven |
| API方式 | REST(JSON) |
| データアクセス | Spring Data JPA(Hibernate) |

## データベース

| 分類 | 内容 |
|---|---|
| DBMS | PostgreSQL |
| データ永続化 | バックエンド経由でPostgreSQLに保存(詳細は[データベース設計書](データベース設計書.md)参照) |
