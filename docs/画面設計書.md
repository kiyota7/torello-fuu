# 画面設計書

タスク管理アプリ「タスクボード」の画面設計。
機能の詳細は [機能一覧.md](機能一覧.md) を参照。

## 画面一覧

本システムは以下の1画面のみで構成される。

| 画面ID | 画面名 | 概要 |
|---|---|---|
| S-01 | ボード画面 | リストとカードを表示・操作する唯一のメイン画面 |

## 画面構成要素

ボード画面は以下の要素で構成する。

- ヘッダー: アプリタイトル、「サンプルに戻す」ボタン
- リスト(複数): リスト名、「優先度順」「期限順」並び替えボタン、カード一覧、「+ カードを追加」ボタン、リスト削除ボタン
- カード(各リストに複数): カード本文、優先度(高・中・低)、期限日、カード削除ボタン
- 「+ リストを追加」: 画面右端に常に表示される、新規リスト作成用の入力欄

## 画面遷移

画面遷移は発生しない(単一画面内でのDOM操作により全ての機能を提供する)。

## 画面イメージ(ワイヤーフレーム)

簡易版のワイヤーフレームを以下に示す。実装時はこのレイアウトを基準とし、
配色・フォント等の詳細デザインは別途モックアップで確認する。

```
+----------------------------------------------------------------------+
|  タスクボード                                    [ サンプルに戻す ]  |
+----------------------------------------------------------------------+
|  To Do        ✕    |  進行中        ✕  |  完了          ✕  |           |
|  [優先度順][期限順] |  [優先度順][期限順]|  [優先度順][期限順]|  + リスト |
|  ------------      |  ------------     |  ------------     |    を     |
|  [カード1     ✕]  |  [カード3     ✕] |  [カード5     ✕] |    追加   |
|  [高 ▾][期限日]    |  [中 ▾][期限日]   |  [低 ▾][期限日]   |           |
|  [カード2     ✕]  |  [カード4     ✕] |                  |           |
|  [中 ▾][期限日]    |  [中 ▾][期限日]   |                  |           |
|  + カードを追加    |  + カードを追加   |  + カードを追加   |           |
+----------------------------------------------------------------------+
```

- カードはマウスドラッグで同一リスト内・他リストの任意の位置へ自由に並び替え・移動できる(F-07)
- リスト名・カード本文はクリックでその場編集ができる(F-02, F-05)
- カードの優先度・期限日は常時表示されるプルダウン/日付入力から直接変更できる(F-10, F-11)
- 各リストの「優先度順」「期限順」ボタンを押すと、そのリスト内のカードのみが一括で並び替わる(F-12)。押した瞬間の整列であり、以後は自由にドラッグして崩せる
- 一番右の「+ リストを追加」列は常に画面右端に固定表示する
