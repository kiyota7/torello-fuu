# データベース設計書

タスク管理アプリ「タスクボード」のデータ設計。
PostgreSQLに以下の構造でテーブルを作成し、バックエンド(Spring Boot + Spring Data JPA)
経由で永続化する。ユーザーごとに専用のボードを持つため、`list`テーブルは`app_user`に
紐づく(単一画面・単一ボード運用のため、Boardテーブル自体は設けない)。

## ER図(エンティティ関連)

```
app_user (1) ── (多) list (1) ── (多) card
```

## テーブル定義

**app_user テーブル**

| カラム名 | 型 | 説明 |
|---|---|---|
| id | BIGSERIAL | 主キー |
| username | VARCHAR(UNIQUE) | ユーザー名(ログインID) |
| password_hash | VARCHAR | ハッシュ化(bcrypt等)されたパスワード |
| created_at | TIMESTAMP | 登録日時 |

**list テーブル**

| カラム名 | 型 | 説明 |
|---|---|---|
| id | BIGSERIAL | 主キー |
| user_id | BIGINT(FK → app_user.id) | 所有するユーザー |
| title | VARCHAR | リスト名(例: To Do, 進行中, 完了) |
| position | INTEGER | リストの表示順(左からの並び順) |

**card テーブル**

| カラム名 | 型 | 説明 |
|---|---|---|
| id | BIGSERIAL | 主キー |
| list_id | BIGINT(FK → list.id) | 所属するリスト |
| text | TEXT | カードの内容(タスク名) |
| priority | VARCHAR | 優先度(「高」「中」「低」の3段階。デフォルトは「中」) |
| due_date | DATE | 期限日(NULL許容。未設定の場合はNULL) |
| position | INTEGER | リスト内での表示順(自由な並び替え・ソート結果を保持) |

## 保存形式

- 保存先: PostgreSQL(バックエンドのSpring Data JPAを介してアクセス)
- 保存タイミング: リスト・カードに対する追加/編集/削除/移動/並び替えが発生する都度、
  フロントエンドからREST API経由でバックエンドに送信し、即時反映する
- 並び替え(F-07, F-12)の結果は`position`カラムに反映し、再取得時も順序を保持する
- 新規登録時(F-13)、そのユーザー用の`list`/`card`をサンプルデータで初期化する
- 全てのlist/card取得・更新APIは、ログイン中のユーザー(セッション)に紐づく`user_id`で
  絞り込み、他ユーザーのデータにはアクセスできないようにする
