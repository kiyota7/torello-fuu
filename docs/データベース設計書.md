# データベース設計書

タスク管理アプリ「タスクボード」のデータ設計。
サーバー・データベースを持たないため、ブラウザのlocalStorage上に
以下の構造のJSONデータとして保持する。

## ER図(エンティティ関連)

```
Board (1) ── (多) List (1) ── (多) Card
```

## データ項目

**List(リスト)**

| 項目名 | 型 | 説明 |
|---|---|---|
| id | 文字列 | リストを一意に識別するID |
| title | 文字列 | リスト名(例: To Do, 進行中, 完了) |
| cards | 配列(Card) | このリストに属するカードの一覧 |

**Card(カード)**

| 項目名 | 型 | 説明 |
|---|---|---|
| id | 文字列 | カードを一意に識別するID |
| text | 文字列 | カードの内容(タスク名) |

## 保存形式

- 保存先: ブラウザのlocalStorage(キー名: `trello-lite-board`)
- 保存タイミング: リスト・カードに対する追加/編集/削除/移動が発生する都度、即時保存する
